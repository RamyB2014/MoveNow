Update Production Overview

To update production on the wells in your project you can use:

  • A production data file you exported from a data vendor, such as Accumap or GeoScout
  • A external data source (IHS, GeoLogic, or corporate data hub for monthly data)
  • The Spreadsheet Import Tool (.xlsx, .xlsm, or .csv files)

Before you update production, review the topics below to ensure your entities are updated according to your expectations.

External Data Source Connection Requirements

To update production using PPDM you need to have an:

  • Oracle client installed
  • IHS or GeoLogic subscription
  • IHS or GeoLogic connection set up in User Options.

To update monthly production using a corporate data hub, you need to specify the vnudl file.

See Configure an External Data Connection.

Configure Import Parameters

Before updating production you must configure the Import Parameters in User Options. The Import Parameters control the behaviour of on-times, ratios, and forecasts. You can configure these parameters before updating production or during the production update.  

Configure External Data Settings

Before updating production you must configure the External Data Settings in User Settings. See User Options: External Data Settings.

The External Data Settings control the type of data and the amount of production history imported. You can configure these options before updating production or during the production update. 

Map Project UWIDs to the Data Source

The well names or UWIDs in your project must match the names or UWIDs in the data source. Wells with names that do not match are not updated.

If the wells in your project were originally imported from the data source you are using to update them, the names should match and you do not need to map the UWIDs. However, the names or UWIDs might be different if:

  • You have used custom well names that a data vendor does not recognize. You can still use custom names, but you must map them to the names in the data source.
  • The production data source you originally used to import the wells and the production data source you are now using come from different data vendors who use different names for the same wells

To determine if the well names are mapped correctly, see Map Data Vendor IDs.
