Create Roles

A role is a group of policies (permissions). By creating roles and then assigning the roles to users, you can quickly apply a consistent set of permissions to several users.

When you assign the Edit Entity or View Entity policies to a role, you must create a filter to specify the entities the role can view or edit. See Create a Filter Query. Rights to view and edit entities are managed using custom fields. See Configure Custom Fields.

To create a role

  1. From the Administration menu, select Security Settings.
  2. On the Roles tab, click Create Role.
  3. Type a role name and click OK.
  4. Click Assign Policy.
  5. In the Select Policies window, select policies to assign to the role. Hold the Ctrl or Shift keys to select multiple policies. If you are assigning the Edit Entity or View Entity policies, see Edit Policies.
  6. Click OK.
