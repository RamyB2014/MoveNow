Balance Entities

When an entity’s opening balance (oil, sales gas, before tax cash flow or before tax NPV) differs from its current balance, the entity is unbalanced. You cannot copy data from one plan to another if the entity is unbalanced. You must balance the entity by creating a change record, which accounts for the difference between the opening and current balances. See Copy Reserves Category Data (and Advanced Copy, or Compare and Promote Reserves Plans.

For an explanation of how change records are displayed and organized on Review | Change Records, see Review | Change Records Overview .

To balance one entity, see Balance One Entity, below.

To balance several entities at once, see Balance Several Entities, below.

Balance One Entity (Single Change Record)

You can balance one entity on Review | Change Records, or by selecting Balance from the Change Management menu, as indicated in the procedure below.

To balance one entity

  1. In the Explorer, select the entity you want to balance.
  2. If you are only creating a change record for one reserves category, select that category (from the Summary window below the Entity Explorer).
  3. Do one of the following:
    4. Select Then
    Review | Change Records. Click Balance (below the change record display window).
    Balance from the Change Management menu. Go to step 4.
  4. In the Balance Entities dialog box, select the appropriate Jurisdiction, Reserves Plan, Reserves Category (All or Current), Change Category, and Change Date.
    5. If you select a Change Date that occurs after the Reference Date, a red exclamation mark () is displayed. You must select a date prior to the Reference Date or the change record will not be included when you post the database.
  5. Add a comment explaining the change and click OK.

Balance Several Entities (Batch Change Record)

To create a change record for several entities

  1. Select the entities for which you want to create a change record (you can filter to the entities or select an appropriate folder level).
  2. If you are only creating a change record for one reserves category, select that category (from the Summary window below the Entity Explorer).
  3. Select Balance from the Change Management menu.
  4. In the Balance Entities dialog box, select the appropriate Jurisdiction, Reserves Plan, Reserves Category (All or Current), Change Category, and Change Date.
    5. If you select a Change Date that occurs after the Reference Date, a red exclamation mark () is displayed. You must select a date prior to the Reference Date or the change record will not be included when you post the database.
  5. Add a comment explaining the change and click OK.

To view change record data for an entity, go to Review | Change Records.
