Create an Empty SQL Server Project

To create a SQL Server or SQL Server - Windows AuthenticationSQL Server: When accessing the database from outside , you will need a password.
SQL Server - Windows Authentication: When accessing the database from outside , Windows Authentication is used. project, you must have SQL Server administrator rights.

Before creating an Oracle or SQL Server project, you need to configure the connection settings in User Options: Connection Settings.

You must use at least SQL Server 2008 for 6 or higher databases.

To create an empty SQL Server (or SQL Server - Windows Authentication) project

  1. On the File menu, click Open.
  2. In the Project Launcher, click Create New Project.
  3. In the Project Name field, type a name.
  4. In File, browse to a location to save the new .vndl file (a -specific connection link for server-based project files).
  5. From the Type list, select SQL Server or SQL Server - Windows Authentication.
  6. In Server, type the name of the server or click to select it from the list.
  7. Click OK.

A new project is created in the location you specified and opens. The new project appears in the Project Launcher the next time you open it.
