Create or Edit an Entity Hierarchy

This procedure explains how to change the folder structure of the entity hierarchy so you can organize your wells in the Entity Explorer in a way that makes your workflow easier.

You can create entity hierarchies to organize your wells according to general well information (such as field, pool, or operator) or by using custom fields).

The entity hierarchies in the project are available in a list above the Entity Explorer.

It is recommended that you always create new hierarchies rather than editing the Default hierarchy. Creating new hierarchies allows for greater flexibility when merging projects and sharing wells with other users.

User and Project Hierarchies

By default, hierarchies you create are User hierarchies. To create a Project Hierarchy, you must select Project Hierarchy in the Edit Hierarchies dialog box.

User hierarchies are available to the user who creates them in any project. If you export project data, User hierarchies are not included in the export.

Project hierarchies are part of the project and only available in that project. If you export project data, Project hierarchies are included in the export.

To create an entity hierarchy

  1. On the View menu, select Entity Hierarchy > Edit Hierarchies.
  2. Do one of the following:
    3. To Do this
    Create an entity hierarchy, Click Add Hierarchy and edit the default name in the Name field.
    Edit an entity hierarchy, In the Select Hierarchy field, select the hierarchy you want to edit.
  3. Move fields from the Available Fields window to the Hierarchy window.
  4. If you want to restrict the hierarchy to the current project, select Project Hierarchy below the Available Fields window.
  5. If you want to use the hierarchy immediately, click Make Active.
    The new hierarchy is applied to your project.
  6. Click OK.
