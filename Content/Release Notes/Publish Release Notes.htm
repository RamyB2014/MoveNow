Publish Release Notes

Release notes should be made available each release as a topic under Release Notes, which is linked to from the home page and as a PDF in the Release Notes topic.

Follow these guidelines for release notes: 

  • Create a new book in the TOC named Release Notes.
  • Create a topic for each release named Version Number/Name Release Notes.
  • Create one topic called Release Notes PDFs and put all PDFs in that topic, with the most recent first.
  • From each Release Notes topic, add a link on the first line to the Release Notes PDFs topic like this: For a PDF version, see Release Notes PDFs.
  • Create a link on the home page named Version Number/NameRelease Notes.
  • When describing a new feature in the release notes, link to the relevant topic or topics in the help.

Use the product-specific Release Notes template to create the Release Notes in Word and then create a PDF from that document.

Save the Release Notes document in the product's SharePoint site.
