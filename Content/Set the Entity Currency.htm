Set the Entity Currency

The entity currency is the currency assigned to the entity. It can be set independently of the project currency and its country’s currency. By default, the currency assigned to an entity without a specified country is the project currency.

If you specify a country for an entity, the entity currency is set to the country currency by default. For example, if the entity is in Canada and the country currency is Canadian dollars, the entity’s currency becomes Default (Canadian dollars), as shown below.

If an entity`s currency equals Default (currency), it inherits the country currency automatically. Therefore, if you change the country currency, all entities in that country using Default (currency) will inherit the new currency. If you set the entity currency to any currency other than Default (currency), the link between the entity currency and the country currency is broken and edits to the country currency will not affect the entity currency, as shown below.

In this example, the entity currency is Default (Canadian dollars) because the country currency for Canada is set to Canadian dollars.

In this example, the entity currency is still Canadian dollars, but it is no longer set as the default. Therefore, if the currency assigned to Canada (country currency) is changed, the entity currency will not change.

To set the entity currency

  1. In the Entity Explorer, right-click the entity you want to edit.
  2. Select Edit Entity.
  3. Select the currency from the Currency list and click OK.
