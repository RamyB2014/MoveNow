Create Custom Results Fields

A custom result field (CRF) enables you to display the data from fiscal regime variables (calculated and trigger variables) and inputs (entity-level and regime-level) in your regimes or Report Designer reports.

When you create a CRF using the procedure below, the CRF inherits the variable’s properties (such as name and unit type). You can also create CRFs independently of the variable and assign your own properties by clicking Custom Result Fields at the bottom of the Fiscal Regimes Editor.

When you create a CRF in a regime or ring fence, it is available in other similar regimes or ring fences (for example, you cannot access a before-tax CRF in an after-tax regime) and in the Report Designer. All CRFs are available for selection when you click the Custom Result list when creating a variable.

To edit CRFs, click the Custom Result Fields button at the bottom of the Fiscal Regime Editor.

To access CRFs in the Report Designer, open the Before Tax Custom Results or After Tax Custom Results folder in the Toolbox. See Create a Report in the Report Designer.

This procedure is performed on the Edit tab in the Fiscal Regimes Editor. To open the editor, click the Tools menu, point to Global Project Data and select Fiscal Regimes.

To create a custom result field

  1. In the Fiscal Regimes Editor, select the variable you want to create a Custom Result Field for.
  2. Under Output, click the Custom result list and then click New.
  3. Keep the default name (copied from the variable name) or enter a new name.
  4. Select a Custom Factor: WI, WI/NRI, or 1.0.
  5. Click OK.
