Hierarchy Volume Reconciliation Report

The Hierarchy Volume Reconciliation Report summarizes and organizes change records according to the Entity Hierarchy and displays the results in eight change record categories. To view this report, you must export it as a .csv or .html file.

You can run the report on the entire hierarchy or on a portion of the hierarchy. You can filter the hierarchy to the desired folders in the Explorer before running the report, but you can also select folder from the hierarchy displayed in the report’s dialog box. Selecting a folder in the dialog box and clicking the Export button generates a report for only that folder.

Economic data from all relevant reserves categories must be current to produce accurate results. If economic data is not current when you run a reserves report, runs economics automatically when you run the report. However, if you want to run reserves economics before running the reserves report, you must run the Reserves Options Scenario on the Reports tab or in the Batch Manager for the entities you want to report on. If you run the Reserves Options Scenario prior to running reserves reports, the reports automatically detect this and will not run it again, resulting in faster display of the report. Running the Default Scenario does not necessarily generate the economic results required to run reserves reports.

See Run Economics before Running Reserves Reports.

To run the Hierarchy Volume Reconciliation report

  1. In the Reserves menu, select Hierarchy Volume Reconciliation.
  2. Select a jurisdiction.
  3. Select the required Report Options and Report Depth.
  4. Select whether you want change records grouped by category or summary type.
  5. Click Export.
  6. Save the report.
