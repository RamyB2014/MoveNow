Create Jurisdictions

When you create a jurisdiction, it is displayed in the scenario list when you select Scenarios on the Reports tab or when you select the Scenarios tab in the Batch Manager. You can also specify jurisdictions when you run reserves or change management reports, balance entities, or post reserves.

To create a jurisdiction

  1. On the Tools menu, point to Global Project Data and select Jurisdictions.
  2. Click Add and enter a name and description.
  3. Select the following options, as required:
    4. Option Description
    Posting Discounting Settings Discount Rates are editable in the Project Options.
    Reserves Categories to Balance Select the reserves category.
    Additional Products in Reserves Reports Select additional products to include in reserves reports.
    Price Deck Select a price deckto use in report calculations.
    Apply Economic Limit Use entity setting: Use each entity’s setting. Enabled: The option is enabled for all entities in the Jurisdiction, regardless of the entity setting. Disabled: The option is disabled for all entities in the Jurisdiction, regardless of the entity setting.
    Economic Limit Calculation Type Select a calculation type.
    Economic Limit Discounting Rate These rates are editable in the Project Options.
    Allow Negative Wedge Results Use project option: Use the setting in Project Options. Enabled: The option is enabled for all entities in the Jurisdiction, regardless of the Project Options setting. Disabled: The option is disabled for all entities in the Jurisdiction, regardless of the Project Options setting.
    Use Overlapping Production History instead of the Forecast Use project option: Use the setting in Project Options.  Enabled: The option is enabled for all entities in the Jurisdiction, regardless of the Project Options setting. Disabled: The option is disabled for all entities in the Jurisdiction, regardless of the Project Options setting.
    Selected Companies Select the companies you want to associate with the Jurisdiction.
    Reference Date This date defaults to the project’s Reference Date, but you can edit it for the Jurisdiction for planning purposes. Jurisdiction report scenarios use the Jurisdiction Reference Date, not the project Reference Date.
    Risk Type Risked: The entity’s Chance of Success and Chance of Occurrence settings are used for report calculations. Unrisked: The entity’s Chance of Success and Chance of Occurrence settings are ignored for report calculations and a value of 100% is used for both COS and COO.
  4. Click OK.
