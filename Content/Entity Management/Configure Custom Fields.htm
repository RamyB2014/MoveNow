Configure Custom Fields

Custom Fields are displayed in the Data View on the Wells and General Economics tab.

Custom Fields are used to label wells with text, dates or numbers. You can then use the custom fields to:

  • Identify new wells imported into a project (using the Import Tag, below)

  • Modify the Entity Hierarchy (see Create or Edit an Entity Hierarchy)

  • Filter to wells (see Create a Filter Query)

  • Assign View and Edit rights to users (see Using Custom Fields for User Security, below).

  • Specify an IP rate that you can select on the Cross Plot Report

  • Enter a working interest factor for working interest roll ups

To create a custom field

  1. Do one of the following:
    • On the Wells and General Economics tab (in the Data View), click Edit Custom Fields.
    • In the Tools menu, point to Global Project Data and click Custom Fields.
  1. In the Custom Fields dialog box, click Add in the bottom left corner. (The Custom Fields dialog box is also accessible from the Tools menu > Global Project Data > Custom Fields.)
  2. Type a Name for the custom field and select a Field Type (Text, Date Number).
  3. Click OK.
  4. If you are creating the field for user security, select Requires administrative privileges to edit values.
  5. To add selectable values for the custom field, click Add beneath the Definition area of the Custom Fields dialog box.
  6. Type a Value and click OK.
  7. Repeat steps 5-7 to add more selectable values, if required.
    8. If the project contains text values in the custom field already, the “update list values” option can be used to automatically populate the text values into the list of selectable values.
  8. Under Select Default List Value for New Entities, select the value you want to use as the default for new entities, if required.
  9. Click OK.

The list of selectable text values can be enforced, by selecting Enforce List Validation. If this option is selected, the user must select a value and cannot enter free text.

You can assign a Unit Type (length, area, mass, etc.) to the values in numeric custom fields. You can also specify which units to use (meters, kilometers, miles, or feet, for example) for metric and imperial measurements.

To export custom fields for use in another project, see Export Project Data.

To import custom fields exported from another project, see Import Project Data.
