Create a Filter Query

Filters are used to find wells that meet a criterion or set of criteria.

When using the filter dialog box:

  • You can use an asterisk (*) as a wild card. For example, if you are searching for wells in the Gull Lake Basal Cantuar pool, but can’t remember the full name, you can enter Pool + Like + Gull Lake*.
  • Some fields require filter keys for accurate results. For example, a filter which specifies Decline=True will return all wells with declines regardless of Reserve Category or Reserve Plan. Therefore, to return all wells that have a decline in the Working Copy of PUD, you must use the Reserve Category and Reserve Plan keys (Decline=True AND Reserve Category=PUD AND Reserve Plan=Working.) For this reason, filter keys are automatically displayed when you select a search field that requires them.
  • If you create entities or edit entity properties that affect the filter results, the results are not automatically updated. Use the Reapply Filter button () to update the results.
When a filter is applied to the project, the Filter Well List icon is red .
