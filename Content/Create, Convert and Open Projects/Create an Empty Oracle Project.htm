Create an Empty Oracle Project

To create an Oracle project, you must have Oracle server administrator rights or you must ask a DBA to create an Oracle user for you. Once the user is created, you can use it to create a new project. You will also require the Oracle password.

Before creating an Oracle or SQL Server project, you need to configure the connection settings in User Options: Connection Settings.

You must use at least Oracle Server 10g and Oracle Client v10 for 6 or higher databases. Oracle 9i databases (and Oracle Client 9) are no longer supported.

To create an empty Oracle project

  1. On the File menu, click Open.
  2. In the Project Launcher, click Create New Project.
  3. In Project Name, type a name.
  4. In File Path, browse to a location to save the .vndl file (a -specific connection link for server-based project files).
  5. From the Type list, select Oracle.
  6. In Server, type the server name or select it from the list.
  7. If you do not want to use the default Oracle User name (schema name), type a new name.
  8. In Password, type the Oracle server password.
  9. Click OK.
  10. Enter your DBA login information.
  11. Click OK.

A new project is created in the location you specified and opens. The new project appears in the Project Launcher the next time you open it.
