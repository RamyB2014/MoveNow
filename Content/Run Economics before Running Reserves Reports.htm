Run Economics before Running Reserves Reports

Economic data from all relevant reserves categories must be current to produce accurate results in any reserves report.

If economic data is not current when you run a reserves report, runs economics automatically when you run the report. However, if you want to run reserves economics before running the reserves report, you must run the appropriate jurisdiction scenario on the Reports tab or in the Batch Manager for the entities you want to report on. 

If you run the jurisdiction prior to running reserves reports, the reports automatically detect this and will not run it again, resulting in faster display of the report.

Running the Default Scenario does not necessarily generate the economic results required to run reserves reports.

To run economics in the Batch Manager before running reserves reports

  1. Click the Batch Manager icon .
  2. In the Batch Manager, click the Entities tab.
  3. Under Entity Selection Mode, click Quick Selection.
  4. Under Entities to report, select the entities and/or folders you want run.
  5. Under Reserves categories, select the reserves categories you want to report on.
  6. On the Scenarios tab, select jurisdiction you want to run.
  7. Do one of the following:
    8. To Go to
    Save the batch for future use Step 8.
    Run the batch Step 9.
  8. Click Save as Batch, enter a name, and click Save.
    The created batch is selected in the batches list.
  9. Click Run.

Once you have saved the batch, in the future you can simply select it from the batch list and run it without having to create it again. For more details, see Create and Run a Batch. (For more details on running an existing batch, scroll down to “To run an existing batch”.)

You can also calculate economics from anywhere in , without generating reports, by clicking the Calculate Economics button in the toolbar (). See Run Economics.
