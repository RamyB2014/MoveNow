Security Policies

The tables below describe the policies in the Security Settings (Administration > Security Settings > Policies).

Administration Policy Allows a user to…
Lock project Lock the project and prevent other users from logging in
Manage archives Create and edit archives
Manage database tables Recalculate cached forecasts, gather database statistics, clear data and delete archives. This modifies the database regardless of any view/edit permissions.
Modify custom fields Modify custom fields found in the Global Project Data
Modify fiscal regimes Modify custom regimes
Modify global project data Modify the global data in the project. This includes the Modify Price Decks, Modify Custom Fields and Modify Fiscal Regimes policies.
Modify plan definitions Modify plan definitions
Modify price decks Modify price decks found in the Global Project Data
Modify project options Modify all project options
Modify security Modify the security settings of the project. This includes adding and/or removing users and/or user groups and assigning policies.
Upgrade project Upgrade an older 6 project to the current version
View global project data View the global data in the project
Entity Access Policy Allows a user to…
Edit entity Edit the specified list of entities. This will also give the user view permission to these entities.
View entity View the specified list of entities
Entity Modification Policy Allows a user to…
Create Entities Gives permission for a user to create new entities. Editing permission will still be taken into account.
Delete Entities Gives permission for a user to delete entities which they have permission to edit.
Export Gives permission for a user to export data to an XML or .vndb file.
Delete all comments Delete comments created by other users on editable wells
Edit in Data View Edit inputs on editable wells in Data View. Wells and Custom Fields properties are editable by all users with Edit Entity.
Elevated user field rights Modify the security protected custom fields on editable wells
Import Update entity data from an external source (PPDM source), flat file, and Spreadsheet Import
Import from External Source Update entities from an external source (PPDM) only
Import XML Import data from an XML file
Reserves administration Post the reserves of the project and import posted records
Review entities Access the review flag
Write to accepted Write results from entity operations to the accepted status
Write to pending Write results from entity operations to the pending status
