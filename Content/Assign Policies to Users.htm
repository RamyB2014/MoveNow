Assign Policies to Users

Project permissions are usually assigned to users through roles, which are groups of policies (permissions). However, you can assign policies (by themselves or in combination with roles) to users when you want to create a unique set of project permissions that you don’t intend to apply to more than one user.

When you assign the Edit Entity or View Entity policies to a user, you must create a filter to specify the entities the user can view or edit. See Create a Filter Query. Rights to view and edit entities are managed using custom fields. See Configure Custom Fields.

To assign policies to a user

  1. From the Administration menu, select Security Settings.
  2. On the Users tab, select a user.
  3. Click Assign Policy.
  4. In the Select Policies window, select policies to assign to the user. Hold the Ctrl or Shift key to select multiple policies. If you are assigning the Edit Entity or View Entity policies, see Edit Policies.
  5. Click OK.
