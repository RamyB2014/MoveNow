Create an Oracle Project from a Production Data File

To create an Oracle project, you must have Oracle server administrator rights or you must ask a DBA to create an Oracle user for you. Once the user is created, you can use it to create a new project. You will also require the Oracle password.

To create an Oracle project you need to configure the connection settings in User Options. See User Options: Connection Settings.

To create an Oracle project from a production data file

  1. On the File menu, click Open.
  2. In the Project Launcher, click Create From Import.
  3. Browse to the production data file and click Open.
  4. In Project Name, type a name.
  5. In File, browse to a location to save the .vndl file (a -specific connection link for server-based project files).
  6. From the Type list, select Oracle.
  7. In Server, type the server name or select it from the list.
  8. If you do not want to use the default Oracle User name, type a new name.
  9. In Password, type the Oracle server password.
  10. Enter your User Name and Password.
  11. Click OK.
  12. In the Import Options dialog box, click Edit to change the options or click Next to continue.
    13. Depending on the last production dates in your data file, you might be asked to set the current month.
  13. Click Finish.
