Create a SQL Server Project from a Production Data File

To create a SQL Server or SQL Server - Windows Authentication projectSQL Server: When accessing the database from outside , you will need a password.
SQL Server - Windows Authentication:  When accessing the database from outside , Windows Authentication is used. , you must have SQL Server administrator rights.

To create a SQL Server project you need to configure the connection settings in User Options: Connection Settings.

You must use at least SQL Server 2008 for 6 or higher databases.

To create a SQL Server (or SQL Server - Windows Authentication) project from a production data file

  1. On the File menu, click Open.
  2. In the Project Launcher, click Create From Import.
  3. Browse to the production data file and click Open.
  4. In Project Name, type a name.
  5. In File, browse to a location to save the .vndl file (a -specific connection link for server-based project files).
  6. From the Type list, select SQL Server or SQL Server - Windows Authentication.
  7. In Server, type the name of the server or click to select it from the list.
  8. Click OK.
  9. In the Import Options dialog box, click Edit to change the options or click Next to continue.

    10. Note: Depending on the last production dates in your data file, you might be asked to set the current month.

  10. Click Finish.

A new project is created in the location you specified and opens. The new project appears in the Project Launcher the next time you open it.
